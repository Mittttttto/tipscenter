• Install Titanium server & Host install patches/license

1. Prior to starting installation, please ensure that there no operating system on the disks. If there is, one could use the following instructions to wipe the disks:

o If NCIR was already installed, then run the following on each server that you want to re-install
$ sudo wipedisk

o If any other OS was installed, then use the following commands for each of the disks and on every server that you want to re-install
$ sudo dd if=/dev/zero of=/dev/sdx bs=512 count=1
where sdx is the disk device. Typically they are sda, sdb, etc.




5 导入之前下载好的NCIR包（CBTS release邮件上可以下载，同时下载对应的lisence和patch 文件），重启

•


2. Boot the first server using the iso image -> Choose 3rd option which is "Text" mode installation


a. Graphical mode doesn't add any value


b. There are three more options available which relate to the all-in-one installation (single node). Instructions for that are available separate here (this is not officially supported yet).

3. Server will be installed and then rebooted automatically

a. Note: It takes ~40 minutes for the installation to finish and server to reboot and come to login prompt


4. At login prompt, use credentials wrsroot/wrsroot for initial login

a. Immediately after logging in using the default password, you will be required to change the password to a new one

5. Set IP address (e.g. controller-0 IP address from the table above) for the OAM port

a. $ sudo ip addr add <OAM_IP_address>/<mask> dev <port>

b. $ sudo ip link set <port> up

c. $ sudo ip route add default via <Gateway_IP_address>

Setting initial IP address for transferring files

$ sudo ip addr add 10.21.14.165/25 dev eth0

$ sudo ip link set eth0 up

$ sudo ip route add default via 10.21.14.129

6. Copy license file & the required patches

a. $ scp username@sourcehost:sourcepath/license.lic /home/wrsroot/license.lic

b. $ scp username@sourcehost:sourcepath/patchfile /home/wrsroot/patchfile

Copying the files

$ scp root@10.21.14.43:/home/installers/WR/TiS-15.09/license.lic /home/wrsroot/license.lic

$ mkdir /home/wrsroot/patches

$ scp root@10.21.14.43:/home/installers/WR/TiS-15.09/patches/*.patch /home/wrsroot/patches

7. Apply the patches and reboot

a. Note: TS_15.12_PATCH_0001.patch is mandatory to be installed in the beginning. All patches can be found from here . If there are other patches available, please install them as well.

b. Note: When downloading the patch files, please remember to Right click on the file name and choose "Save As" or "Save Target As"
$ cd /home/wrsroot/patches
$ sudo sw-patch upload *.patch
$ sudo sw-patch apply --all
$ sudo sw-patch install-local

$ sudo sw-patch query (for check patch installation status)
$ sudo reboot

$ sudo sw-patch host-install --force controller-0 (if you cannot install patches after computer configure, you can try force install)

Applying patches

d. If other patches are applied later, then they need to be applied separately to individual servers. Patching instructions can be found from here .

• Installing second controller node

1. Power on the second controller node. If there is no operating system installed, based on the BIOS settings, most servers will eventually go for network boot.

2. A couple of minutes after powering on, one should be able to see a un-configured node, where hostname and personality are shown as "None". One can see the same using the CLI on active controller. One can also see similar information in horizon dashboard under Inventory / Hosts.
